TCP

TCP protocol

TCP는 transport layer에서 응용 프로그램과 네트워크 동작 사이의 매개체로서 사용되며, 포트번호를 이용한 process to process 통신을 제공함.

stream delivery

TCP는 송신측에서 수신측으로 바이트 스트림 형태로 데이터를 전송하며 송신측에서 데이터를 생산하는 속도와 수신측에서 데이터를 소비하는 속도가 다를 경우를 대비해 송신버퍼와 수신버퍼를 설정해 오류를 제어함. 이때 일련의 바이트 스트림을 세그먼트라고 하는 패킷으로 그룹화해서 전송함.

TCP는 데이터를 동시에 양방향으로 전송 및 수신하는 전 이중(full duplex)서비스를 제공함. 즉, 각 노드에는 송신버퍼와 수신버퍼가 동시에 존재함.

Segment

TCP의 패킷인 세그먼트는 20~60바이트의 크기를 갖는 헤더와 응용 프로그램이 전송하는 데이터로 구성되어 있음.

  • Source port address: 발신지 포트 주소
  • Destination port address: 목적지 포트 주소
  • Sequence number: 순서번호
    세그먼트 데이터의 첫번째 바이트에 부여된 번호. 첫 sequence number는 연결설정단계에서 난수로 설정됨.
  • Acknowledgement number: 확인응답번호
    수신 노드가 상대 노드로부터 수신하고자 하는 바이트(cumulative ack). 만약 수신측이 seq번호가 x인 세그먼트를 정상적으로 수신했다면 수신측은 송신측으로 ack번호로 x+1을 보냄.
  • HLEN: 헤더 길이
    4비트이므로 0~15를 나타낼 수 있는데, HLEN의 값에 4를 곱하면 헤더의 길이임
  • Reserved: 예약
  • 제어필드
    • URG: urgent pointer is valid
    • ACK: acknowledgement is valid
    • PSH: request for push
    • RST: reset the connection
    • SYN: synchronize sequence numbers
    • FIN: terminate the connection
  • Window size: 윈도우 크기
    flow control, congestion control에 사용되는 rwnd
  • Checksum: 체크섬
  • Urgent pointer: 긴급포인터

TCP 연결

연결 시작

TCP에서의 연결설정은 3-way handshake라고 하며 아래와 같은 순서로 이루어짐.

  1. 서버는 클라이언트로부터의 연결을 수락하기 위해 수동개방함(passive open)
  2. 클라이언트는 수동개방중인 서버에게 연결을 시작할 것임을 알리는 능동개방을 실행(active open)
  3. 클라이언트는 연결에 필요한 seq번호를 동기화 하기 위해 첫번째 세그먼트로 SYN flag가 1로 설정된 세그먼트를 전송함. 이때 seq번호는 임의의 난수로 지정함().
  4. SYN 세그먼트를 받은 서버는 그에 대한 응답으로 SYN과 ACK flag가 1로 설정된 SYN+ACK 세그먼트를 전송. 이때 서버에서 클라이언트로 데이터를 보낼 때 사용될 seq번호와 flow control에 사용될 rwnd를 헤더에 포함해 초기화함.
  5. SYN+ACK 세그먼트를 받은 클라이언트는 그에 대한 응답으로 ACK 세그먼트를 전송.

3개의 세그먼트를 통해 연결이 establish되므로 3-way handshake라고 불림

데이터 전송

양측의 연결이 establish된 이후에는 양 방향으로 데이터가 전송될 수 있음. 위 사진에서 처럼 client가 8001~9000, 9001~10000번째 데이터와 server에게 15001번째 데이터를 요청하는 ack번호가 담긴 2개의 request를 보냈을 때 이 두 패킷에 대한 response로 서버는 client로부터 10000번까지의 데이터를 받았고 다음 데이터로 10001번째 데이터를 달라는 ack번호와 client가 요청했던 15001~17000번째 데이터가 담긴 패킷으로 응답함.

연결 종료

연결 종료에는 3-way handshake와 half-close가 가능한 4-way handshake 두가지 옵션이 있다.

3-way handshake를 통한 연결 종료는 다음과 같이 동작한다.

  1. client에서 FIN flag가 1로 설정된 FIN 세그먼트를 전송함.
  2. FIN 세그먼트를 수신한 server는 FIN+ACK 세그먼트를 전송함.
  3. FIN+ACK 세그먼트를 수신한 client는 FIN 세그먼트를 받았음을 알려주기 위해 ACK 세그먼트를 전송해 연결을 종료함.

4-way handshake를 통한 연결 종료는 다음과 같이 동작한다.

  1. client에서 FIN flag가 1로 설정된 FIN 세그먼트를 전송함.
  2. FIN 세그먼트를 수신한 server는 이에 대한 응답으로 ACK 세그먼트를 전송. client가 처음 보냈던 FIN 세그먼트에 대한 응답을 받음으로서 client의 sending buffer는 닫히고 receiving buffer만 살아있는 상태가 됨
  3. 이후 server에서 client로 연결 종료 전까지 보내야 할 데이터들을 계속 보냄. (half-close 상태)
  4. server에서 client로 모든 데이터를 보냈다면 비로소 server에서 client로 모든 데이터를 보냈음을 의미하는 FIN 세그먼트를 전송.
  5. FIN 세그먼트를 수신한 client는 FIN 세그먼트를 받았음을 알려주기 위해 ACK 세그먼트를 전송해 연결을 종료함.

3-way handshake와 4-way handshake는 FIN 세그먼트를 받은 server가 FIN+ACK 세그먼트 혹은 ACK 세그먼트를 보내는지에 차이점이 있음.

코드

server

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <signal.h>
#include <sys/wait.h>
#include <arpa/inet.h>
#include <sys/socket.h>

#define BUF_SIZE 30
void error_handling(char *message);
void read_childproc(int sig);

int main(int argc, char *argv[]) {
	int serv_sock, clnt_sock;
	struct sockaddr_in serv_adr, clnt_adr;

	pid_t pid;
	struct sigaction act;
	socklen_t adr_sz;
	int str_len, state;
	char buf[BUF_SIZE];
	if(argc!=2) {
		printf("Usage : %s <port>\n", argv[0]);
		exit(1);
	}

	act.sa_handler=read_childproc;
	sigemptyset(&act.sa_mask);
	act.sa_flags=0;
	state=sigaction(SIGCHLD, &act, 0);
	serv_sock=socket(PF_INET, SOCK_STREAM, 0);
	memset(&serv_adr, 0, sizeof(serv_adr));
	serv_adr.sin_family=AF_INET;
	serv_adr.sin_addr.s_addr=htonl(INADDR_ANY);
	serv_adr.sin_port=htons(atoi(argv[1]));

	if(bind(serv_sock, (struct sockaddr*) &serv_adr, sizeof(serv_adr))==-1)
		error_handling("bind() error");
	if(listen(serv_sock, 5)==-1)
		error_handling("listen() error");

	while(1) {
		adr_sz=sizeof(clnt_adr);
		clnt_sock=accept(serv_sock, (struct sockaddr*)&clnt_adr, &adr_sz);
		if(clnt_sock==-1)
			continue;
		else
			puts("new client connected...");
		pid=fork();
		if(pid==-1) {
			close(clnt_sock);
			continue;
		}
		if(pid==0) {
			close(serv_sock);
			while((str_len=read(clnt_sock, buf, BUF_SIZE))!=0)
				write(clnt_sock, buf, str_len);

			close(clnt_sock);
			puts("client disconnected...");
			return 0;
		}
		else
			close(clnt_sock);
	}
	close(serv_sock);
	return 0;
}

void read_childproc(int sig) {
	pid_t pid;
	int status;
	pid=waitpid(-1, &status, WNOHANG);
	printf("removed proc id: %d \n", pid);
}
void error_handling(char *message) {
	fputs(message, stderr);
	fputc('\n', stderr);
	exit(1);
}

client

#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <arpa/inet.h>
#include <sys/socket.h>

#define BUF_SIZE 30
void error_handling(char *message);
void read_routine(int sock, char *buf);
void write_routine(int sock, char *buf);

int main(int argc, char *argv[]) {
	int sock;
	struct sockaddr_in addr;
	char buf[BUF_SIZE];

	sock = socket(PF_INET, SOCK_STREAM, 0);

	addr.sin_family = AF_INET;
	addr.sin_addr.s_addr = inet_addr(argv[1]);
	addr.sin_port = htons(atoi(argv[2]));
	if(connect(sock, (struct sockaddr*)&addr, sizeof(addr)) == -1) {
		error_handling("connect error");
	}

	pid_t pid = fork();
	if(pid == 0) {
		read_routine(sock, buf);
	}
	else {
		write_routine(sock, buf);
	}
	close(sock);
	return 0;
}

void read_routine(int sock, char *buf) {
	int str_len;
	while((str_len = read(sock, buf, BUF_SIZE)) != 0) {
		buf[str_len] = 0;
		printf("message from server: %s", buf);
	}
}
void write_routine(int sock, char *buf) {
	while(1) {
		fgets(buf, BUF_SIZE, stdin);
		if(!strcmp(buf, "q\n")) {
			shutdown(sock, SHUT_WR);
			return;
		}
		write(sock, buf, strlen(buf));
	}
}
void error_handling(char *message) {
	fputs(message, stderr);
	fputc('\n', stderr);
	exit(1);
}
